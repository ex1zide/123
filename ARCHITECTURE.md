# 🏛️ ARCHITECTURE.md: Стандарт Zero Trust

Данный документ описывает фундаментальные архитектурные решения в LegalHelp KZ. Нашим главным приоритетом является абсолютная безопасность данных пользователей (Zero Trust) и безупречная реактивность (Golden Laptop Standard).

---

## 1. Слой данных и Состояние (Riverpod + Firestore)
Мы полностью отказались от использования ненадежных локальных ключей (`SharedPreferences`) для хранения критических состояний (таких как активные подписки пользователя).

*   **Истина в реальном времени (SSOT):** Все `subscription` и `RBAC` (Role-Based Access Control) данные пользователя синхронизируются через реактивные стримы `Firestore`.
*   **Riverpod Global State:** Мы используем кодогенерируемые `@riverpod` аннотации для создания графа зависимостей. Это исключает утечки стейта и гарантирует, что при истечении подписки или изменении роли `Dashboard` мгновенно отреагирует (например, отобразив Paywall при попытке доступа к PRO).

## 2. Безопасность: Zero Trust и Cloud Functions
Клиентское приложение **не содержит** секретных ключей для сторонних AI API (Google Gemini).

*   **Только HttpsCallable:** Приложение делает только подписанные `HttpsCallable` запросы к Firebase Cloud Functions (написанным на TypeScript).
*   **Серверная Валидация:** Cloud Functions проверяют `context.auth` на сервере. Только валидные, авторизованные пользователи могут начать сессию **ИИ-аудита договоров** или **Legal RAG (Retrieval-Augmented Generation)** чата. Ключ Gemini (`GEMINI_API_KEY`) хранится в `runWith({ secrets: ["GEMINI_API_KEY"] })`.
*   **Firestore Rules:** Строгие правила `firestore.rules` гарантируют изоляцию данных (`request.auth.uid == resource.data.uid`), а также позволяют доступ в `/admin` панель только пользователям с кастомным `isAdmin` флагом.

## 3. Сейф Документов: Биометрия и Шифрование (Offline-First)
Политика строгой конфиденциальности требует защиты локальных данных в случае кражи девайса.

*   **Ключи шифрования:** Мы генерируем защищенный ключ базы данных и надежно прячем его через `flutter_secure_storage` (подкрепленный Android Keystore / Apple Keychain).
*   **Биометрический шлюз:** Проход к локальным документам защищен через пакет `local_auth` (Face ID / Touch ID). Без биометрии документы не дешифруются в памяти устройства.
*   **Reactive Isar Database:** Сами PDF-сканы и результаты OCR-анализа складируются в локальной `Isar` базе данных. Она быстрая, транзакционная и обеспечивает мгновенный поиск без лагов в UI-треде.
